深入理解Android虚拟机读书笔记

Posted on 2016-07-15 Edited on 2019-11-29 In Android Views: Valine:

《深入理解Android虚拟机》 本书的阅读笔记，整体来说这本书只适合通读，入门了解。通过对比阅读，发现很多内容不够严谨，同时内容安排上有拼凑之嫌。如果你和我一样，因为名字想阅读这本书，建议在这本书上花费的时间不要超过一天。

主要内容

本书内容共13章，其中第1章[Android系统介绍]，第2章[Android系统结构]直接跳过，第7章[生命周期管理]明显拼凑，和Dalvik虚拟机关系不大，并且这章内容属于初级Android开发掌握的，第11章[JNI接口]介绍了基础的JNI编程，了解NDK的可以跳过。删减之后，剩下的内容就是这些

  • 虚拟机概述
  • 编译和调试Dalvik
  • Dalvik 虚拟机运作流程
  • Dex优化和安全管理
  • 内存分配策略
  • 垃圾收集
  • 线程管理
  • 异常管理

虚拟机概述

在没有接触和了解虚拟机之前，总觉得虚拟机是很高深，很‘底层’，很容易望文生义。虚拟机其实就是一套解释规范的执行者，解析规范的格式的文件，执行文件中的内容。这章概述主要介绍JVM虚拟机和Dalvik虚拟机的结构，原理。JVM部分建议参考JAVA虚拟机规范SE7。通过阅读本章和规范，了解虚拟机结构和关键术语。

JVM虚拟机

Java虚拟机的数据类型

  • byte
  • short
  • int
  • long
  • float
  • double
  • char
  • object 对一个Java object 的4个字节引用
  • returnAddress 4字节，用于 jsr/ret/jsr-w/ret-w指令

Java虚拟机的体系结构

  • 指令集
  • 寄存器
  • 栈
  • 无用单元收集堆(Garbage-collected-heap)
  • 方法区域

Java虚拟机基于栈的，这点是和Dalvik虚拟机主要区别之一。

Dalvik虚拟机

dx 工具用来讲 .class 文件转换为 dex 文件。转换过程中，会进行优化，重复的字符串和其它常数在dex中只会保留一份。一个未压缩的dex通常比已经压缩的jar小一点。

Dalvik 和 Java虚拟机的差异：

  • Dalvik 早期没有JIT，2.2版本开始支持JIT技术
  • Dalvik 使用自己的bytecode ,并非适用java的bytecode
  • Dalvik 基于寄存器(Register),JVM 基于栈(Stack)
  • Dalvik 通过Zygote 进行 Class Preloading ,Zygote 会完成虚拟机的初始化，这也是雨JVM虚拟机不同之处。
